## 目次

### 1. [Skills システム](./01-skills.md)
AI エージェントに新しい機能を与える仕組み

- **概要:** マークダウンファイルや MCP サーバーを通じてエージェントに新しい能力を追加
- **主要特徴:**
  - 静的コンテキストファイル（SKILL.md）
  - 動的ツール接続（MCP サーバー）
  - 進段的開示によるコンテキスト管理
- **ハンズオン:** カスタムスキルの作成、インストール、使用

### 2. [Subagent（サブエージェント）](./02-subagent.md)
タスクの委任と並列処理パターン

- **概要:** メインエージェントがサブタスクを専門化されたサブエージェントに委任
- **主要特徴:**
  - 目標駆動型の委任
  - 並列実行
  - 結果の統合
- **ハンズオン:** サブエージェントの作成、委任パターンの実装

### 3. [AGENTS.md](./03-agents.md.md)
AI エージェント向けのプロジェクト仕様書

- **概要:** プロジェクトのコンテキストと指示をエージェントに提供するオープン標準
- **主要特徴:**
  - プロジェクト概要
  - 開発環境設定
  - コーディング規約
  - ワークフロー指示
- **ハンズオン:** AGENTS.md の作成、ベストプラクティス

### 4. [OpenClaw](./04-openclaw.md)
オープンソース自律 AI エージェント

- **概要:** ローカルで動作し、自律的にタスクを実行するオープンソース AI エージェント
- **主要特徴:**
  - ローカルゲートウェイ
  - 永続メモリ（SOUL.md、MEMORY.md）
  - スキルシステム（ClawHub）
  - 24/7 動作
- **ハンズオン:** インストール、設定、カスタムスキル作成

### 5. [Agent Teams](./05-agent-teams.md)
Claude のマルチエージェントオーケストレーション

- **概要:** 複数の専門エージェントを並列で実行するオーケストレーションシステム
- **主要特徴:**
  - リーダーエージェントとサブエージェントモデル
  - 並列実行
  - 独立した観測可能性
- **ハンズオン:** Agent Teams の起動、タスク分割、結果統合

### 6. [Context Fork](./06-context-fork.md)
コンテキストのフォークと分岐パターン

- **概要:** 異なるコンテキストでエージェントを分岐させて並列処理
- **主要特徴:**
  - コンテキストの分離
  - 並列探索
  - 結果のマージ
- **ハンズオン:** コンテキストフォークの実装、パターン適用

### 7. [Git Worktree](./07-git-worktree.md)
Git Worktree を使った並列エージェント実行

- **概要:** Git worktree 機能を使用して複数のエージェントを並列で実行
- **主要特徴:**
  - 分離されたワークスペース
  - ブランチベースの隔離
  - コミットとマージのワークフロー
- **ハンズオン:** Worktree の作成、並列エージェント実行、結果マージ
